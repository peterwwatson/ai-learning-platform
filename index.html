Project Hub

Central place for the AI Learning Platform project. All vision, architecture, strategy, current work, and processes in one place.

Purpose

A real, high-quality learning system to help Peter master modern AI (especially agentic systems, loops, and practical use) using only trusted sources and clear progression.

Core Approach

Multi-agent collaboration with strict quality gates, real content only, and transparent reporting. The system itself demonstrates good agentic practices.

Vision Document

Vision: A trustworthy personal learning system that gives clear understanding of the latest AI developments (2026 focus), best ways to use AI effectively, and actionable skills for real work.

Non-negotiables: Real content only • Quality review on everything • Progressive layered learning • Real (not fake) progress tracking • Continual updates from trusted sources.

Full detailed version available in Vision_and_Strategy_Document.md

System Architecture

Layers

  • Database Layer: SQLite or structured files to manage resources, tasks, progress, and summaries.
  • Agent Layer: Multi-agent team running disciplined loops with quality gates.
  • Interface Layer: Accessible webpage that reads from the database/files and displays current state.
  • Documentation Layer: This Hub + key Markdown files (Vision, Work Logs, Reports).

Development Strategy

We use a strict multi-agent loop with clear handoffs. Every piece of work goes through Planner → Executor → Reviewer → Reporter.

Focus is always on real learning value for Peter, not building infrastructure for its own sake. The database supports the learning system — it is not the main product.

Ideal Development Loop

  1. 1. Leader / Architect — Reviews vision and current state. Defines or refines the next piece of work.
  2. 2. Planner — Breaks the work into clear tasks and records them in the database.
  3. 3. Executor — Completes the tasks (research, summarisation, coding, updating files, etc.).
  4. 4. Reviewer — Checks quality, accuracy, and alignment with vision. Rejects or requests fixes if needed.
  5. 5. Reporter — Produces a clear status report and updates the Hub / work log.
  6. 6. Handoff — Records what was done, current state, and what is next for the next agent or session.

This loop ensures quality and traceability at every step.

Agent Team & Roles

Leader / Vision Owner

Sets direction, approves major changes, maintains the Vision document.

Work Record: Updates Vision & Strategy Document and produces high-level status reports.

Architect

Designs system structure, data models, and processes.

Work Record: Maintains Architecture and Process documents.

Researcher / Curator

Finds high-quality new content from trusted sources.

Work Record: Adds entries to Resources database with source, quality score, and summary.

Analyst

Processes long content (transcribes videos, summarises threads).

Work Record: Creates structured summaries stored in the database.

Educator

Turns content into clear, layered learning material with practical application steps.

Work Record: Creates or updates learning modules and explanations.

Reviewer / QA

Quality gate — checks accuracy, clarity, actionability, and vision alignment.

Work Record: Adds review notes and approval status to work items.

Reporter

Produces clear status reports after each loop.

Work Record: Updates Current State and adds reports to the Hub.

Handoff Protocol

Between Agents (within a session):

  • Each agent appends a short work log entry (what was done, decisions, open items).
  • Explicit handoff note: “Handing off to Reviewer with the following status…”

Between Sessions:

  • New session must first read the latest Vision_and_Strategy_Document.md and Current_State.md (or this Hub).
  • Start with a summary of the latest work and next priorities.
  • Reference the Project Hub for all documents.

Key Processes

Content Addition Process: Researcher → Analyst (if needed) → Educator → Reviewer → Librarian (add to DB) → Reporter

Session Start Protocol: Always begin by reading the latest Vision document and Current State. Confirm alignment before starting work.

Quality Gate: Nothing reaches Peter without passing the Reviewer.
